<PlanStep n={2} title="Detailed proposal">
  <h4>What we're proposing</h4>
  <p>Adopt React 18 + TypeScript + Vite. Build a thin Strangler Fig router that owns the top-level URL space and delegates to either a React page or a sandboxed iframe hosting the existing jQuery page. Migrate components in 4 risk-ordered waves. Decommission the jQuery bundle after Wave 4.</p>

  <h4>Architecture</h4>
  <Card title="High-level">
    <div style={{ fontFamily: "var(--font-mono)", fontSize: 12, lineHeight: 1.7 }}>
      <div>┌────────────────────────────────────────────┐</div>
      <div>│  React 18 Router (new, owns URL space)      │</div>
      <div>├────────────────┬───────────────────────────┤</div>
      <div>│  React page    │  iframe → jQuery page       │</div>
      <div>│  (TypeScript)  │  (legacy, sandboxed)        │</div>
      <div>├────────────────┴───────────────────────────┤</div>
      <div>│  postMessage bridge (auth, navigation)     │</div>
      <div>└────────────────────────────────────────────┘</div>
    </div>
  </Card>

  <h4>Acceptance criteria</h4>
  <ul>
    <li><input type="checkbox" data-progress /> New features ship in React from day 1</li>
    <li><input type="checkbox" data-progress /> Existing jQuery pages work unchanged during migration</li>
    <li><input type="checkbox" data-progress /> Auth + nav work across the iframe bridge</li>
    <li><input type="checkbox" data-progress /> Bundle size stays under 200 KB gzipped (current React shell: 87 KB)</li>
    <li><input type="checkbox" data-progress /> TS strict mode catches ≥80% of bugs at compile time</li>
  </ul>
</PlanStep>
